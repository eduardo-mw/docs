This guide walks you through setting up Application Performance Monitoring (APM) on a .NET application. These instructions can also be found on the Installation page in your Middleware Account. View example code <a href="https://github.com/middleware-labs/demo-apm/tree/master/dotnet"> here. </a>

# Prerequisites

<Steps>

  <Step title="MW Host Agent">
   Middleware Host Agent (MW Agent). To install the MW Agent, see our <a href="https://docs.middleware.io/docs/agent-installation/overview"> installation guide</a>. 
  </Step>

  <Step title=".NET Version">
    `.NET version 7+`. Check your .NET version with the following command.
    ```bash Shell
    dotnet --version
    ```
  </Step>

</Steps>

# Install

<Accordion title="Step 1: .NET Binary Installation">

Download the bash script.
```bash Shell
curl -sSfL https://install.middleware.io/apm/dotnet/v1.0.0-rc.1/scripts/mw-dotnet-auto-install.sh -O
```
Install core files.
```bash Shell
sh ./mw-dotnet-auto-install.sh
rm -rf mw-dotnet-auto-install.sh
```
Enable execution for the instrumentation script.
```bash Shell
chmod +x $HOME/.mw-dotnet-auto/instrument.sh
```
</Accordion>

<Accordion title="Step 2: Setup Middleware .NET Project">

* Download Latest [Middleware.dll](https://github.com/middleware-labs/dotnet-plugin/releases) and add the following code in the .csproj file..

```xml XML
<ItemGroup>
  <Reference Include="Middleware">
    <HintPath>path\to\Middleware.dll</HintPath>
  </Reference>
</ItemGroup>
```
</Accordion>

<Accordion title="Step 3: Build and Run .NET Application">

  ```bash Shell
  MW_API_KEY={MW_API_KEY} \
    . $HOME/.mw-dotnet-auto/instrument.sh && \
    OTEL_SERVICE_NAME="{APM-SERVICE-NAME}" \
    dotnet path/to/YourApplication.dll
  ```
</Accordion>
